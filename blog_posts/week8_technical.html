What is it...

Structured Query Language (SQL) Injection is one of the many web attack mechanisms to steal data from originations, suppliers, employees, and customers!! Really.. from YOU!!! It is one of the most common application attack techniques used today. It is an attack that takes advantage of improper coding of the web application that allows hackers to "inject" SQL commands into, say, login in forms to allow them access into your life! (or the database). It is so popular that when I did started my research I typed in SQL Injection and what do you think the one of the first 3 links were? Yeah t-shirts, there are even t-shirts for your kid...


Web applications (web apps) and Databases (db).
Db are a pretty much central to our everyday life on the web. They store data needed for web sites to deliver specific content to visitors and produce information to customers, suppliers, employees, (YOU!!!) or update, change information given by us, the user! Lets say the user visits a site that needs a user name and password. When the user submits their details a SQL query is generated and submitted to the data base for verification, once the information is verified, access is allowed! Since the site needs to be public, how else would the public access it, the security in place needs to allow public web traffic to communicate with the web app, the web app has open access to the db, it needs to change, update, add too, I'm sure the list goes on. SQL is the language the computer uses to perform those actions. In fact, as far as I know, SQL is the only way web apps to interact with database. Firewalls and other same type barriers wont work because the site needs to be public.
