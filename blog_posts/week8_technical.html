Regular ExpressionsSeptember 28, 2014


What is it...

A regexps holds a regular expression, used to match a pattern against strings. Regexps are created using the /../ and %r{...} listerals, and by the Regexp::new constructor. Regular expressions (regexps) are patterns which describe the contents of a string. They're used for testing whether a string contains a given pattern or extracting the portions that match. They are created with the /pat/ and %r{pat} literals or the Regexp.new constructor.
ruby-doc.org


Regular expression is a very powerful tool that we can use. I learned of regex when I first started using Ruby. In the first excersises that we all did in Ruby, Devbootcamp asked us not to use them but it sure could have made life a little easier. I am going to try and explain and give just a few examples. I'm not going to get deep into this, there have been hole books written just on regular expression.2

Regex quick reference

Lets say that I want to print a string to the screen

regex
001
sentence = "Yo, what is your name?"
002
puts sentence

result => Yo, what is your name?

That really isn't a polite way to ask for someone's name. Maybe we should say something like, Excuse me Sir, what is your name? There are a few ways to do this but I think one great way is regex.

regex
001
sentence.gsub(/^../, "Excuse me sir")

result => Excuse me sir, what is your name?

Lets talk a little bit about what we did here. #gsub is a cool method that helps us look for things and replace them. Regex always start with AND end with a / forward slash. What is inside the forward slashes is the magic. The first symbol is a caret. ^ This is an anchor that tells Ruby to start at the beginning of the string. If we wanted to change something at the end of the string we could use a dollar sign like this sentence.gsub(/....$/, "Last name") and it would have changed the word name to last name. After the caret you see two dots. These dots represent the number of charters. notice in my last example I used 4 dots then the dollar sign that is because name is 4 charters long. get it?


One more, I could go on for ever with this stuff... Lets say I want to extract certain things from a string, like numbers, or pattern of things. I'm going to make a simple variable called... hmm... football, hey why not?

regex
001
football = "I am 45 years old and my dog is 21 years old!"

result => I am 45 years old and my dog is 21 years old!

So lets say that I want to extract all the numbers with in this variable. The symbol that we want to use is \d. That means scan for any digit (d).

regex
001
football.scan(/\d/) { |x| puts x }

result => 4

result => 5

result => 2

result => 1

Well, that's really not what we want right? we wanted our code to return the numbers 45 and 21. See ruby doesn't know if we wanted a 4 and 5 or if we want 45. So we have to add another symbol the + sign, this tells Ruby to return all joining digits... like this

regex
001
football.scan(/\d+/) { |x| puts x }

result => 45

result => 21

For my last example, lets use a challenge from our past. We needed to find and print words containing a given letter. This was the code I wrote to begin with:

regex
001
i_want_pets = ["I", "want", 3, "pets", "but", "only", "have", 2]
002
def my_array_finding_method(source, thing_to_find)
003
str = source.map(&:to_s)
004
finding_lettter=str.select{|letter|letter.downcase.include?(thing_to_find)}
005
end
006
p my_array_finding_method(i_want_pets, "e")

result => ["pets", "have"]

what was to be returned was all the words containing the letter e witch would be pets and have The code I wrote was cool and it worked but maybe a better way might have been like this.

regex
001
i_want_pets = ["I", "want", 3, "pets", "but", "only", "have", 2]
002
def my_array_finding_method(source, thing_to_find)
003
source.select { |x| x =~ /[#{thing_to_find}]/ }
004
end
005
p my_array_finding_method(i_want_pets, "e")

result => ["pets", "have"]

Couple new things here, we have the ~ called the Tilda. In Ruby this basically means when ever you want to compare to regular expression. [ ] The brackets are called charter classes or groups. So basically what this method is doing is iterating over the array of strings when ever it comes across the given letter return that 'group' of character. Nice huh? I think it is pretty great!
