The Well-Grounded Rubyist, second edition1
Like an array, a hash is a collection of objects. A hash consists of key/value pairs, where any key and any value can be any Ruby object. Hashes let you perform lookup operations based on keys. In addition to simple key-based value retrieval, you can also perform more complex filtering and selection operations.
Programming Ruby 1.9&2.0(4th edition): The Pragmatic Programmers' Guide (pickaxe)2
Hashes (sometimes known as associative arrays, maps, or dictionaries) are similar to arrays in that they are indexed collections of object references. However, while you index arrays with integers, you index a hash with objects of any type: symbols, strings, reqular expressions, and so on.
