Responsive Design and Fluid Layouts

The Fundamental Concept Behind Adaptable Design for Web Development

By: Taha Siddiqui & Troy Leach

Responsive Design2

Responsive Web Design was the answer to the problem of "changing landscapes". In essence, due to the fact that the internet was now readily accessible on different mediums such as smart phones and tablets, users expected to be able to browse the web with as much ease on these new devices as they did before on their personal computers. For this reason, web developers started creating various versions for each website to accomodate the new devices. In retrospect, due to the fact that the technology industry is continously evolving it did not seem like a permanent fix to continue to create two versions of every site. Thus Responsive Design was born.

Fluid Layouts1

The fundamental concept behind responsive design lies in the fluid gird. The standard before hand-held web browers were readily accessible, and when everyone viewed websites primary on a computer-sized screen, was that websites were created using fixed wisth layouts. The web pages would be designed using pixels as the primary unit of measure to modify changes to the appearance of the website. This process would create a standard layout which at the time was acceptable due to a lack of various screen resolutions.

The idea behind creating Fluid grids is that, by using percentages as opposed to a pixel count for the new unit of measure, the website can become adaptable to the size of the screen. When used conciously, the site can be designed so that all of the elements on the page will "resize their widths in relation to one another."
